# Документація по розробленому веб-застосунку

## Призначення застосунку

 **MyWebApp (Notes Service)** - це веб-сервіс для створення, зберігання та перегляду текстових нотаток.
 Застосунок побудований на мові Go з використанням принципів чистої архітектури.<br> Він підтримує роботу як з реальною реляційною базою даних MariaDB, так і в режимі in-memory.
 
## Як налаштувати середовище для розробки та тестування
- Завантажити [Go](https://go.dev/dl/) 
- Завантажити [MariaDB](https://mariadb.org/download/)
- Клонувати репозиторій та завантажити залежності:
```
git clone https://github.com/blvckwhiteee/devops-labs-2026.git

cd devops-labs-2026

go mod tidy
```
- Ініціалізувати базу даних:
```
mysql -u root -p < scripts/init_db.sql
```
- Запустити міграції та застосунок:
```
go run ./cmd/migrate -db="user:password@tcp(127.0.0.1:3306)/mywebapp_db?parseTime=true"

go run ./cmd/mywebapp -port=3000 -db="user:password@tcp(127.0.0.1:3306)/mywebapp_db?parseTime=true"
```

## Документація по API-ендпоінтах
### Ендпоїнти бізнес-логіки
Усі ендпоінти підтримують заголовок **Accept (application/json або text/html)**
- `GET /` (Кореневий ендпоінт) - повертає HTML-сторінку зі списком усіх доступних ендпоінтів бізнес-логіки
- `GET /notes` - повертає список об'єктів із полями`id, title` всіх створених нотаток
- `POST /notes` - створює нову нотатку
  - Тіло запиту у форматі JSON: ```{
"title": "Назва нотатки",
"content": "Текст нотатки"
} ```
  - Відповідь із статусом ```HTTP 201 Created``` та поверненням `ID` новоствореної нотатки
- `GET /notes/{id}` - отримує повну інформацію про конкретну нотатку за `id` у форматі `{
    "id": "id",
    "title": "Назва",
    "content": "Текст нотатки",
    "created_at": "Час створення"
}`
### Внутрішні ендпоїнти 
При спробі звернення до них через зовнішню адресу (коли застосунок розгорнуто) буде отримана помилка 404 через Reverse Proxy (nginx). Ці ендпоїнти доступні лише всередині локальної мережі.
- `GET /health/alive` - завжди повертає HTTP 200 з вмістом OK
- `GET /health/ready` — повертає HTTP 200 з вмістом OK, якщо сервіс успішно підключився до БД і готовий обробляти запити, інакше повертає HTTP 500
