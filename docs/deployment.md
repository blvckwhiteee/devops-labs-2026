# Документацію по розгортанню

Рекомендується за основу взяти [Ubuntu **Server** 24.04.4 LTS ](https://ubuntu.com/download/server)

## Встановлення та налаштування віртуального оточення
### Рекомендовані ресурси ВМ
- RAM - 2048 MB
- CPU - 1 ядро
- Диск - 25 GB

Перед встановленням ОС необхідно зайти у налаштування **мережі**:
- Тип підключення - NAT
- Переадресування порту (configure port forwarding) та створюємо два правила:
  - Головний порт: 2222. Гостьовий порт: 22. 
  - Головний порт: 8080. Гостьовий порт: 80. 
### Встановлення віртуального оточення
- Виконуємо звичайну встановку, але :
  - Ім'я: **ubuntu**
  - Пароль: **123456**
- Також необхідно встановити OpenSSH server
### Підключення через SSH
```
ssh ubuntu@127.0.0.1 -p 2222

// yes -> 123456

sudo apt install git

git clone https://github.com/blvckwhiteee/devops-labs-2026.git
cd devops-labs-2026'

sudo bash scripts/setup.sh 
// 123456
```

## Тестування розгорнутого застосунку
### Перевірка коретної роботи API
- Переходимо за адресою ```http://127.0.0.1:8080/```. Якщо застосунок було правильно розгорнуто, то побачимо список ендпоїнтів.

- Для перевірки роботи nginx переходимо за адресою ```http://127.0.0.1:8080/health/alive```. Працездатність ендпоїнту можемо перевірити в терміналі
### Перевірка ролей 
- Якщо ми вийдемо за домомогою `exit` із сесії у терміналі, то при `ssh ubuntu@127.0.0.1 -p 2222` отримаємо `Permission denied, please try again.`
- Перевірка на прикладі teacher:
```
ssh teacher@127.0.0.1 -p 2222
// 12345678
// -> 123456

sudo systemctl status mywebapp.service

curl http://127.0.0.1:3000/health/ready
```
